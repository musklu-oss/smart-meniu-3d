## Directory Purposes

**`assets/images/`:**
- Purpose: Source imagery for AR target tracking
- Contains: `target.jpg` (used as AR marker), `marker-printabil.jpg` (physical print version for testing)
- Key files: `target.jpg` — this is what the camera must see to trigger AR

**`assets/models/`:**
- Purpose: 3D dish models, one per dish SKU, in two formats
- Contains: `.glb` for browser (Three.js), `.usdz` for iOS native AR (Safari / `model-viewer`)
- Naming pattern: `snake_case_dish_name.glb` / `PascalCase_dish_name.usdz`
- Key files: `ballotin_of_saithe_with_herbs.glb` — only dish currently wired in `index.html`

**`assets/targets/`:**
- Purpose: Compiled MindAR binary target data (`.mind` files)
- Contains: Empty at repo level — `targets.mind` generated at runtime or by `compile-target.mjs`
- Generated: Yes — do not hand-edit
- Committed: No (empty directory committed, generated file excluded)

**`.planning/codebase/`:**
- Purpose: Architecture documentation for GSD tooling
- Contains: `ARCHITECTURE.md`, `STRUCTURE.md`
- Generated: Yes (by GSD map-codebase agent)
- Committed: Yes

## Key File Locations

**Entry Points:**
- `index.html`: Browser app — entire AR experience in one file
- `compile-target.mjs`: CLI tool — run with `node compile-target.mjs` to compile `target.jpg` → `targets.mind`

**Configuration:**
- `package.json`: Node.js dependencies for dev tools only (no runtime bundler)

**Core Logic:**
- `index.html` `<script>` block (lines 56–172): IndexedDB cache, MindAR init, Three.js render loop
- `compile-target.mjs`: Target compilation pipeline (Node.js path only)

**Styling:**
- `index.html` `<style>` block (lines 7–36): AR app styles — loading screen, AR container, product overlay
- `style.css`: Product card / detail page design system — separate visual language, uses CSS custom properties

**3D Assets:**
- `assets/models/*.glb`: Browser-compatible 3D models
- `assets/models/*.usdz`: iOS native AR models

**AR Target:**
- `assets/images/target.jpg`: Source image compiled into tracking data
- `assets/images/marker-printabil.jpg`: Print this to test AR tracking physically

## Naming Conventions

**Files:**
- HTML/CSS/JS scripts: lowercase, hyphenated (`index.html`, `compile-target.mjs`, `style.css`)
- 3D models (`.glb`): lowercase snake_case matching dish name (`ballotin_of_saithe_with_herbs.glb`)
- 3D models (`.usdz`): PascalCase (`Ballotin_of_saithe_with_herbs.usdz`) — iOS convention
- Images: lowercase hyphenated (`target.jpg`, `marker-printabil.jpg`)
- Compiled targets: lowercase (`targets.mind`)

**CSS classes (in `style.css`):**
- BEM-lite, hyphenated: `.product-info`, `.product-name`, `.product-price`, `.product-badge`, `.ar-button`
- Layout wrappers: `.page`, `.viewer-wrap`, `.header`

**DOM IDs (in `index.html`):**
- Hyphenated: `#loading`, `#ar-container`, `#product-overlay`, `#progress-bar`, `#progress-fill`, `#progress-text`

**JS functions (in `index.html` inline script):**
- camelCase verbs: `openDB`, `getCached`, `saveCache`, `setProgress`, `getTargetBuffer`, `startAR`

## Where to Add New Code

**New dish / AR item:**
1. Add GLB model to `assets/models/new_dish_name.glb`
2. Add USDZ model to `assets/models/New_dish_name.usdz` (if iOS AR needed)
3. In `index.html`: change `loader.load('assets/models/...')` path, update `#product-overlay` text (name, description, price)
4. Replace `assets/images/target.jpg` with new marker image (or add multi-target support)
5. Bump `CACHE_KEY` string (e.g. `target-v2`) to force cache invalidation after target change

**New UI state / screen:**
- Add new `<div>` to `index.html` body with `display: none`
- Add inline styles to the `<style>` block in `index.html`
- Toggle via `element.style.display` in the inline script

**Product detail page (separate from AR view):**
- Implement using `style.css` design tokens — it defines `.product-info`, `.product-name`, `.product-price`, `.product-badge`, `.ar-button`, `.viewer-wrap`
- Create a new `product.html` that links `style.css` and uses `<model-viewer>` web component

**New offline CLI tool:**
- Add as `.mjs` file at project root, following `compile-target.mjs` pattern
- Use `canvas` and `mind-ar` from existing `package.json` deps
